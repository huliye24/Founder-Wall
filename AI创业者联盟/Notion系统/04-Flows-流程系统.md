# Flows Database - 流程系统

> **版本**：v1.0  
> **核心理念**：让信息在系统中流动起来  
> **更新时间**：2024年1月

---

## 一、字段定义

### 基础字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Name | 标题 | Flow名称 |
| Type | 选择 | 流程类型 |
| Description | 文本 | 流程描述 |
| Version | 文本 | 版本号 |

### 流程定义

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Input Type | 多选 | 输入类型 |
| Process Steps | 文本 | 处理步骤（有序列表） |
| Process Agents | 关联 | 涉及的Agent |
| Output Type | 多选 | 输出类型 |
| Trigger | 选择 | 触发方式 |

### 运行字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Status | 选择 | 状态 |
| Frequency | 选择 | 运行频率 |
| Last Run | 日期 | 最后运行 |
| Next Run | 日期 | 下次运行 |
| Run Count | 数字 | 运行次数 |
| Success Rate | 百分比 | 成功率 |

### 绩效字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Efficiency | 数字 | 效率评分 1-10 |
| Quality | 数字 | 质量评分 1-10 |
| Cost | 文本 | 运行成本 |

### 关联字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Related Inputs | 关联 | 此Flow处理过的输入 |
| Related Outputs | 关联 | 此Flow产生的输出 |
| Related Projects | 关联 | 关联项目 |

## 五、自定义Flow创建指南

### 创建新Flow步骤

```
1️⃣ 识别场景
   ├─ 什么样的输入？
   ├─ 需要什么样的处理？
   └─ 期望什么样的输出？

2️⃣ 定义步骤
   ├─ 列出处理步骤（3-7步最佳）
   ├─ 分配每步的Agent
   ├─ 确定步骤间的依赖关系

3️⃣ 设计模板
   ├─ 输入模板
   ├─ 输出模板
   └─ 决策点模板

4️⃣ 设置触发
   ├─ 手动触发（快速测试）
   ├─ 事件触发（自动化）
   └─ 时间触发（定期执行）

5️⃣ 测试运行
   ├─ 运行3-5次
   ├─ 记录问题
   └─ 优化调整

6️⃣ 正式上线
   ├─ 设置Status = 运行中
   ├─ 监控统计数据
   └─ 持续优化
```

### Flow设计原则

```
✅ 简洁性：3-7步最佳，不要过度复杂
✅ 单一职责：每个Flow只做一件事
✅ 清晰输入：明确什么情况下触发
✅ 明确输出：定义清晰的产出物
✅ Agent匹配：让合适的Agent做合适的事
✅ 可观测：设计统计指标，便于监控

❌ 避免
├─ 步骤太多（超过10步考虑拆分）
├─ 过于通用（失去针对性）
├─ 依赖复杂（步骤间强耦合）
└─ 无法自动化（需要太多人工介入）
```
