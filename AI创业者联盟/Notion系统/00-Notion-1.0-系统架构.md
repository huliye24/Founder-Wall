> **版本**：1.0 MVP

## 三、核心数据库设计

### 1️⃣ Memory Database（记忆库）

**本质**：系统的"脑"，所有信息最终沉淀的地方

#### 数据库字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Title | 标题 | 记忆简述 |
| Type | 选择 | 灵感 / 对话 / 结论 / 观察 / 决策 / 问题 |
| Source | 选择 | ChatGPT / Claude / Gemini / 人 / 自己 |
| Content | 文本 | 具体内容 |
| Tags | 多选 | 自由标签 |
| Related Agent | 关联 | 关联的AI/人节点 |
| Related Project | 关联 | 关联的项目 |
| Importance | 数字 | 重要度 1-5 |
| Status | 选择 | 原始 / 处理中 / 已归档 / 待行动 |
| Created | 日期 | 创建时间 |
| Flow Status | 选择 | 未流动 / 流动中 / 已流动 |

#### 视图

```
📊 Memory 记忆库
├── 📨 全部记忆（画廊视图）
├── 🔥 今天（24小时内的活跃记忆）
├── ⚡ 待处理（Status = 处理中）
├── 🏷️ 按标签（分组视图）
├── 🤖 按来源（分组视图）
└── 📈 重要度排行（排序视图）
```

#### 快速操作模板

```
┌────────────────────────────────────────────┐
│ 🧠 新建记忆                                    │
├────────────────────────────────────────────┤
│ Title: [标题]                                │
│ Type: [选择]                                 │
│ Source: [来源]                               │
│ Content:                                     │
│ ┌────────────────────────────────────────┐  │
│ │                                        │  │
│ │                                        │  │
│ └────────────────────────────────────────┘  │
│ Tags: [标签1] [标签2]                        │
│ Importance: ★★★★☆                          │
│                                            │
│ → 是否需要流动？                             │
│   □ 分配给 Agent                            │
│   □ 关联到 Project                         │
│   □ 创建新 Flow                            │
└────────────────────────────────────────────┘
```

### 2️⃣ Agents Database（AI/人节点库）

**本质**：管理"认知节点"，而非管理人

#### 数据库字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Name | 标题 | Agent名字 |
| Avatar | 媒体 | 头像/图标 |
| Type | 选择 | AI / 人 / 混合 |
| Role | 多选 | 策略 / 编程 / 研究 / 创作 / 执行 / 顾问 |
| Capability | 文本 | 核心能力描述 |
| Specialty | 多选 | 专长领域 |
| Status | 选择 | 在线 / 休眠 / 实验中 / 离线 |
| Connected Projects | 关联 | 参与的项目 |
| Memory Links | 关联 | 处理的记忆 |
| Performance | 数字 | 表现评分 1-10 |
| Task Count | 数字 | 完成任务数 |
| Last Active | 日期 | 最后活跃时间 |
| Prompt Template | 文本 | Agent提示词模板 |

#### Agent 角色定义

| 角色 | 职责 | 输入 | 输出 |
|------|------|------|------|
| 🧠 策略Agent | 分析、规划、决策支持 | 问题/数据 | 策略建议/方案 |
| 💡 创意Agent | 头脑风暴、创新、灵感激发 | 话题/问题 | 创意点子/方案 |
| 💻 技术Agent | 编程、实现、技术方案 | 需求/问题 | 代码/技术方案 |
| 📊 研究Agent | 信息收集、分析、报告 | 主题 | 研究报告/数据 |
| 🎯 执行Agent | 任务分解、执行、跟进 | 目标 | 执行结果/报告 |
| 💰 投资Agent | 评估、尽调、投资建议 | 项目 | 投资分析/建议 |
| ⚖️ 顾问Agent | 提供专业建议、审核 | 问题 | 专业意见/建议 |

#### 视图

```
🤖 Agents 节点库
├── 👥 全部节点（画廊视图）
├── 🟢 在线Agent
├── 🔬 实验中
├── 📊 按角色（分组视图）
├── 🏆 表现排行
└── ⚡ 活跃排行
```

### 3️⃣ Projects Database（项目系统）

**本质**：所有创业/实验的容器

#### 数据库字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Name | 标题 | 项目名称 |
| Cover | 媒体 | 项目封面 |
| Stage | 选择 | 构思 / 验证 / 开发 / 测试 / 发布 / 完成 / 暂停 / 归档 |
| Goal | 文本 | 核心目标 |
| Description | 文本 | 项目描述 |
| Related Agents | 关联 | 参与的Agent |
| Related Memories | 关联 | 关联的记忆 |
| Status | 选择 | 活跃 / 暂停 / 完成 / 归档 |
| Priority | 选择 | P0 / P1 / P2 / P3 |
| Progress | 数字 | 进度 0-100% |
| Start Date | 日期 | 开始时间 |
| Target Date | 日期 | 目标完成时间 |
| Milestones | 关系 | 关联的任务组 |
| Outputs | 文本 | 项目产出 |
| Learnings | 文本 | 经验教训 |

#### 项目阶段定义

```
Stage 1: 构思 (Idea)
├── 定义问题
├── 市场调研
└── 组建团队

Stage 2: 验证 (Validate)
├── MVP开发
├── 用户测试
└── 假设验证

Stage 3: 开发 (Build)
├── 产品开发
├── 迭代优化
└── 质量保证

Stage 4: 测试 (Test)
├── Beta测试
├── 反馈收集
└── 优化调整

Stage 5: 发布 (Launch)
├── 市场推广
├── 用户获取
└── 数据监控

Stage 6: 完成 (Complete)
├── 复盘总结
├── 收益评估
└── 经验沉淀
```

#### 视图

```
📦 Projects 项目中心
├── 🎯 全部项目（看板视图，按Stage分组）
├── 🔥 活跃项目
├── 📊 进行中
├── ✅ 已完成
├── ⏸️ 暂停项目
└── 🏆 重点项目
```

### 4️⃣ Flows Database（流程系统）

**本质**：定义信息如何流动，类似NodeNet协议雏形

#### 数据库字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Name | 标题 | Flow名称 |
| Type | 选择 | 信息流 / 工作流 / 决策流 / 协作流 |
| Input Type | 选择 | Memory / Agent / Project / External |
| Process Agent | 关联 | 处理的Agent |
| Intermediate Steps | 文本 | 中间步骤 |
| Output Type | 选择 | Memory / Decision / Project / Action |
| Status | 选择 | 设计中 / 测试中 / 运行中 / 已归档 |
| Trigger | 文本 | 触发条件 |
| Frequency | 选择 | 单次 / 每日 / 每周 / 按需 |
| Last Run | 日期 | 最后运行时间 |
| Performance | 数字 | 效率评分 1-10 |
| Notes | 文本 | 备注说明 |

#### 预置 Flow 模板

```
┌─────────────────────────────────────────────────────────────┐
│ 🔁 Flow 1: 灵感处理流                                         │
├─────────────────────────────────────────────────────────────┤
│ Input: 新灵感 (Memory)                                        │
│ ↓                                                             │
│ Process: [创意Agent] 分析灵感                                 │
│ ↓                                                             │
│ Decision: 是否值得深挖？                                       │
│   ├─ Yes → 创建项目关联                                       │
│   └─ No → 归档至知识库                                        │
│ ↓                                                             │
│ Output: 项目 (Project) / 知识 (Knowledge)                     │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│ 🔁 Flow 2: 问题解决流                                         │
├─────────────────────────────────────────────────────────────┤
│ Input: 问题 (Memory)                                          │
│ ↓                                                             │
│ Process: [研究Agent] 收集信息                                  │
│ ↓                                                             │
│ Process: [策略Agent] 分析问题                                 │
│ ↓                                                             │
│ Process: [技术Agent] 提出方案                                 │
│ ↓                                                             │
│ Output: 解决方案 (Memory) + 任务 (Project)                    │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│ 🔁 Flow 3: 项目孵化流                                         │
├─────────────────────────────────────────────────────────────┤
│ Input: 项目创意 (Project)                                      │
│ ↓                                                             │
│ Stage 1: [研究Agent] 市场验证                                  │
│ ↓                                                             │
│ Stage 2: [策略Agent] + [技术Agent] 产品规划                   │
│ ↓                                                             │
│ Stage 3: [执行Agent] 开发执行                                  │
│ ↓                                                             │
│ Stage 4: [投资Agent] 融资准备                                  │
│ ↓                                                             │
│ Output: 成熟项目 + 退出                                        │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│ 🔁 Flow 4: 每日信息流                                         │
├─────────────────────────────────────────────────────────────┤
│ Morning: [研究Agent] 行业日报                                  │
│ ↓                                                             │
│ Afternoon: [策略Agent] 信息整合                                │
│ ↓                                                             │
│ Evening: [创意Agent] 灵感激发                                  │
│ ↓                                                             │
│ Output: 每日简报 (Memory)                                      │
└─────────────────────────────────────────────────────────────┘
```

#### 视图

```
🔁 Flows 流程系统
├── ⚡ 全部Flow
├── 🧪 测试中
├── 🚀 运行中
├── 📈 效率排行
└── 🔧 自定义Flow
```

## 四、核心运行逻辑

### 信息流动循环

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        🔄 AI联盟核心循环                                     │
└─────────────────────────────────────────────────────────────────────────────┘

                              ┌───────────────┐
                              │               │
                              │    Memory     │
                              │    (输入)      │
                              │               │
                              └───────┬───────┘
                                      │
                                      ▼
                              ┌───────────────┐
                              │               │
                              │  Agent 处理    │
                              │   分析/创造    │
                              │               │
                              └───────┬───────┘
                                      │
                                      ▼
                              ┌───────────────┐
                              │               │
                              │   Project     │
                              │   生成/执行    │
                              │               │
                              └───────┬───────┘
                                      │
                                      ▼
                              ┌───────────────┐
                              │    Output     │
                              │   决策/行动    │
                              │               │
                              └───────┬───────┘
                                      │
                                      ▼
                              ┌───────────────┐
                              │               │
                              │    Memory     │
                              │   (沉淀)      │
                              │               │
                              └───────────────┘

                     "信息不流动 = 系统死亡"
```

### 每日工作流

```
🌅 Morning (8:00 - 9:00)
├── 📊 查看 Dashboard
├── 🧠 处理 Inbox 记忆
├── 🔄 启动 每日信息流
└── 📋 确认今日任务

🕐 Working (9:00 - 18:00)
├── 🔥 执行 Active Projects
├── 💡 触发 Flow 处理问题
├── 🤖 与 Agents 协作
└── 📝 沉淀新知识

🌙 Evening (18:00 - 19:00)
├── 📊 更新项目进度
├── 🧠 整理今日记忆
├── 🔮 规划明日工作
└── 📈 周报生成
```

## 七、关键原则

### ❌ 避免的坑

| 坑 | 表现 | 后果 |
|----|------|------|
| 漂亮笔记系统 | 只有收藏，没有流动 | 系统死亡 |
| 静态存储 | 信息堆积，无结构 | 难以复用 |
| 过度设计 | 字段太多，无法坚持 | 放弃使用 |
| 单一视角 | 只有自己，没有Agent | 失去协作 |

### ✅ 正确的做法

| 原则 | 说明 |
|------|------|
| 流动优先 | 每个信息都要考虑下一步去哪 |
| Agent中心 | 通过Agent连接人与信息 |
| 最小MVP | 先跑起来，再迭代优化 |
| 定期复盘 | 每周检视系统运行状况 |
