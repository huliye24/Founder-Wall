## 一、字段定义

### 核心字段（必须）

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| Title | 标题 | 记忆简述（30字内） | "AI Agent将颠覆SaaS定价模型" |
| Type | 选择 | 信息类型 | 灵感/对话/结论/观察/决策/问题/数据 |
| Source | 选择 | 信息来源 | ChatGPT/Claude/Gemini/人类/自己/播客/文章 |
| Content | 文本 | 具体内容 | 支持富文本、图片、代码块 |
| Tags | 多选 | 自由标签 | #AI #创业 #投资 #技术 |
| Importance | 数字 | 重要度 1-5 | ★★★★★ = 5 |

### 关联字段（关键）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Related Agent | 关联 | 处理或生成此记忆的Agent |
| Related Project | 关联 | 关联的项目 |
| Related Knowledge | 关联 | 相关知识库条目 |

### 状态字段（管理）

| 字段名 | 类型 | 选项 | 说明 |
|--------|------|------|------|
| Status | 选择 | 原始/处理中/已归档/待行动 | 当前状态 |
| Flow Status | 选择 | 未流动/流动中/已流动 | 是否进入Flow |
| Next Action | 文本 | - | 下一步行动描述 |
| Due Date | 日期 | - | 行动截止日期 |

### 元数据字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Created | 日期（自动） | 创建时间 |
| Updated | 日期（自动） | 更新时间 |
| Processed By | 人 | 处理过此记忆的人 |
| View Count | 数字 | 查看次数 |

## 五、使用指南

### 每日习惯

```
🌅 Morning (5分钟)
1. 打开 Memory Inbox
2. 处理昨天的"待处理"记忆
3. 分配给对应的 Agent 或 Project

🕐 Working (实时)
1. 任何灵感 → 立即记录到 Memory
2. 重要对话 → 归档到 Memory
3. 关键结论 → 沉淀到 Memory

🌙 Evening (5分钟)
1. 回顾今天的记忆
2. 更新 Flow Status
3. 设置明天的 Next Action
```

### 信息进入 Flow

```
判断标准：这条信息是否需要下一步处理？

YES → 设置 Flow Status = 流动中
      选择目标 Flow
      分配给 Agent

NO → 设置 Flow Status = 已流动
     或直接归档
```

### 归档规则

```
可归档：
✓ 已完成 Action
✓ 已整合进知识库
✓ 过期无效信息
✓ 完全无价值的记录

保留原始：
✓ 重要决策及理由
✓ 关键对话记录
✓ 灵感来源追溯
```

## 六、快捷键 & 技巧

### Notion 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Cmd+N` | 新建页面 |
| `/table` | 插入数据库 |
| `/toggle` | 折叠块 |
| `Cmd+\`` | 代码块 |
| `@` | 关联/提及 |

### 效率技巧

1. **快速记录**：手机端用 Notion 快捷方式
2. **语音转文字**：用语音输入再整理
3. **截图归档**：重要信息截图存入 Memory
4. **定时整理**：每周末花30分钟整理本周记忆
