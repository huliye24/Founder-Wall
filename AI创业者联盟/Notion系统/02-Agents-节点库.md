## 一、字段定义

### 基础字段

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| Name | 标题 | Agent/人的名称 | "策略官 Alpha" |
| Avatar | 媒体 | 头像/图标 | 🤖 / 👤 |
| Type | 选择 | 节点类型 | AI / 人 / 混合 |
| Role | 多选 | 核心角色 | 策略/编程/研究/创作/执行/顾问/投资 |
| Status | 选择 | 在线状态 | 在线/休眠/实验中/离线 |

### 能力描述

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Capability | 文本 | 核心能力详细描述 |
| Specialty | 多选 | 专长领域 |
| Strengths | 文本 | 优势 |
| Weaknesses | 文本 | 局限 |
| Best For | 文本 | 最适合的场景 |

### 协作字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Connected Projects | 关联 | 参与的项目 |
| Memory Links | 关联 | 处理过的记忆 |
| Flows | 关联 | 参与的流程 |
| Collaboration Partners | 关联 | 常协作的其他Agent |

### 绩效字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Task Count | 数字 | 完成任务总数 |
| Success Rate | 百分比 | 成功率 |
| Avg Response Time | 文本 | 平均响应时间 |
| Performance Score | 数字 | 综合评分 1-10 |
| Last Active | 日期 | 最后活跃时间 |

### 模板字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Prompt Template | 文本 | Agent提示词模板 |
| Output Format | 选择 | 输出格式偏好 |
| Preferences | 文本 | 使用偏好/习惯 |

## 二、Agent 角色体系

### 🧠 策略官 (Strategy Agent)

```
职责：
- 分析宏观趋势
- 制定战略规划
- 评估重大决策
- 资源配置建议

输入：问题/数据/趋势
输出：战略建议/方案/分析报告

Prompt 模板：
"你是一位经验丰富的战略顾问。
背景：{context}
问题：{question}
请从以下几个维度分析：
1. 机会与威胁
2. 优势与劣势
3. 行动方案
4. 风险评估"

擅长领域：商业战略、投资分析、竞争格局
配合Agent：研究官、投资官
```

### 💡 创意官 (Creative Agent)

```
职责：
- 头脑风暴
- 创新方案设计
- 问题重新定义
- 灵感激发

输入：话题/问题/约束
输出：创意点子/方案/灵感

Prompt 模板：
"你是一位极具创意的设计师/创业者。
背景：{context}
挑战：{challenge}
约束：{constraints}
请提供10个创意方向，包括：
1. 核心创意
2. 实现思路
3. 潜在价值
4. 风险点"

擅长领域：产品创意、营销创意、体验设计
配合Agent：策略官、技术官
```

### 💻 技术官 (Technical Agent)

```
职责：
- 技术方案设计
- 代码审查
- 架构规划
- 技术问题解决

输入：需求/问题/代码
输出：技术方案/代码/架构图

Prompt 模板：
"你是一位资深的AI工程师。
技术栈：Python, TensorFlow, PyTorch, LangChain
问题：{problem}
要求：{requirements}
请提供：
1. 技术方案
2. 实现步骤
3. 代码示例
4. 注意事项"

擅长领域：AI开发、全栈技术、架构设计
配合Agent：策略官、创意官
```

### 📊 研究官 (Research Agent)

```
职责：
- 信息收集整理
- 市场调研
- 竞品分析
- 数据分析

输入：研究主题/问题
输出：研究报告/数据/图表

Prompt 模板：
"你是一位专业的研究分析师。
目标：{objective}
范围：{scope}
深度：{depth}
请提供：
1. 执行摘要
2. 关键发现
3. 数据支撑
4. 参考来源"

擅长领域：市场研究、行业分析、数据挖掘
配合Agent：策略官、创意官
```

### 🎯 执行官 (Execution Agent)

```
职责：
- 任务分解
- 进度跟踪
- 执行协调
- 结果交付

输入：目标/任务/截止日期
输出：执行计划/进度报告/交付物

Prompt 模板：
"你是一位高效的项目经理。
目标：{goal}
资源：{resources}
截止：{deadline}
请提供：
1. 任务分解（WBS）
2. 时间表
3. 风险点
4. 里程碑"

擅长领域：项目管理、运营执行、流程优化
配合Agent：所有Agent
```

### 💰 投资官 (Investment Agent)

```
职责：
- 项目评估
- 尽职调查
- 投资建议
- 退出规划

输入：项目BP/财务数据
输出：投资分析/尽调报告/投资建议

Prompt 模板：
"你是一位经验丰富的投资人。
关注阶段：Pre-A 到 B轮
领域：AI、科技
项目：{project}
请评估：
1. 团队
2. 市场
3. 产品
4. 商业模式
5. 财务预测
6. 风险
7. 投资建议"

擅长领域：投资评估、财务分析、资本市场
配合Agent：策略官、研究官
```

### ⚖️ 顾问官 (Advisor Agent)

```
职责：
- 专业意见
- 方案审核
- 合规检查
- 经验分享

输入：方案/问题
输出：专业意见/建议/风险点

Prompt 模板：
"你是一位在{领域}有20年经验的专家。
请审核以下方案：
{content}
提供：
1. 专业意见
2. 潜在风险
3. 改进建议
4. 类似案例"

擅长领域：法律、财务、技术、运营
配合Agent：根据领域选择
```

## 七、使用指南

### 何时调用哪个Agent

| 场景 | 推荐Agent | 原因 |
|------|-----------|------|
| 不知道该做什么 | 策略官 | 帮你分析方向 |
| 缺乏创意 | 创意官 | 头脑风暴 |
| 技术问题 | 技术官 | 专业解决方案 |
| 需要做调研 | 研究官 | 系统性收集信息 |
| 执行效率低 | 执行官 | 任务分解和跟踪 |
| 项目融资 | 投资官 | 专业投资视角 |
| 专业问题 | 顾问官 | 领域专家意见 |

### Agent 使用习惯

```
1. 明确问题
   在调用Agent前，先清楚自己要问什么

2. 提供背景
   给Agent足够的上下文信息

3. 指定输出格式
   告诉他你想要什么格式的回复

4. 多Agent协作
   复杂问题可以用多个Agent分工处理

5. 记录结果
   将Agent的输出存入Memory，形成知识沉淀
```
