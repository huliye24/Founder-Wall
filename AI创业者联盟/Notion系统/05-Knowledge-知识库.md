## 一、字段定义

### 基础字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Title | 标题 | 知识标题 |
| Topic | 选择 | 知识主题 |
| Type | 选择 | 知识类型 |
| Cover | 媒体 | 封面图片 |
| Summary | 文本 | 一句话摘要 |

### 内容字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Content | 文本 | 详细内容 |
| Key Points | 文本 | 关键要点 |
| Tags | 多选 | 标签 |
| Difficulty | 选择 | 难度级别 |
| Status | 选择 | 状态 |

### 来源字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Source | 文本 | 来源URL/出处 |
| Author | 人 | 作者/贡献者 |
| Source Agent | 关联 | 整理的Agent |
| Created | 日期 | 创建时间 |
| Updated | 日期 | 更新时间 |

### 关联字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Related Projects | 关联 | 关联项目 |
| Related Memories | 关联 | 相关记忆 |
| Related Knowledge | 关联 | 相关知识 |

### 互动字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Views | 数字 | 阅读次数 |
| Likes | 数字 | 点赞数 |
| Rating | 数字 | 评分 1-5 |
| Comments | 评论 | 评论数 |

## 二、知识分类体系

### Topic 分类

```
📚 Knowledge Topics

├─ AI/ML（机器学习、深度学习、LLM）

├─ 商业计划书模板
├─ 项目管理模板
├─ 日常办公模板
└─ 复盘总结模板
```

| 词典 | 术语解释 | "Web3术语词典" |

### Difficulty 难度

| Level | 说明 | 受众 |
|-------|------|------|

## 四、知识创作流程

### 知识沉淀标准流程

```
🧠 来源识别
├─ Memory中的重要结论
├─ 项目中的经验教训
├─ AI对话中的有价值信息
└─ 外部学习的知识

    ↓

📝 知识整理
├─ 撰写标题和摘要
├─ 提炼关键要点
├─ 撰写详细内容
├─ 添加实践技巧

    ↓

🏷️ 分类打标
├─ 选择Topic
├─ 选择Type
├─ 选择Difficulty
└─ 添加Tags

    ↓

🔗 关联建立
├─ 关联Related Projects
├─ 关联Related Memories
├─ 关联Related Knowledge

    ↓

📤 发布分享
├─ 设置Status = 已发布
├─ 添加到相应Knowledge Base视图
└─ 通知相关人

    ↓

📊 持续优化
├─ 根据Views/Likes优化
├─ 定期更新内容
└─ 收集反馈改进
```

## 九、知识搜索策略

### 快速找到知识

```
1️⃣ 按Topic浏览
   → 知道大概方向时

2️⃣ 按Tag搜索
   → 有明确关键词时

3️⃣ 按Related搜索
   → 在项目/记忆页面时

4️⃣ 全局搜索
   → 知道标题关键词时
```

### 知识关联网络

```
Memory → 触发学习 → Knowledge沉淀
              ↓
         Project → 应用知识 → 输出成果
              ↓
         经验 → 复盘 → Knowledge

Knowledge ↔ Knowledge（横向关联）
```
