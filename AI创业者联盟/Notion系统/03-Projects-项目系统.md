## 一、字段定义

### 基础字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Name | 标题 | 项目名称 |
| Cover | 媒体 | 项目封面图 |
| Stage | 选择 | 项目阶段 |
| Goal | 文本 | 核心目标（一句话） |
| Description | 文本 | 项目详细描述 |
| Status | 选择 | 项目状态 |

### 管理字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Priority | 选择 | 优先级 P0/P1/P2/P3 |
| Progress | 数字 | 进度 0-100% |
| Stage Progress | 数字 | 当前阶段进度 0-100% |
| Health | 选择 | 健康度 良好/一般/危险 |
| Start Date | 日期 | 开始日期 |
| Target Date | 日期 | 目标完成日期 |
| Actual End | 日期 | 实际完成日期 |

### 协作字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Owner | 人 | 项目负责人 |
| Related Agents | 关联 | 参与的AI Agent |
| Team Members | 人 | 人工团队成员 |
| Related Memories | 关联 | 相关记忆 |
| Related Knowledge | 关联 | 关联知识 |

### 资源字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Budget | 数字 | 预算（元） |
| Spent | 数字 | 已花费 |
| Resources | 文本 | 所需资源 |
| Deliverables | 文本 | 交付物清单 |

### 成果字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Outputs | 文本 | 项目产出 |
| Learnings | 文本 | 经验教训 |
| Metrics | 文本 | 关键指标 |
| Reflections | 文本 | 复盘总结 |

## 二、项目阶段定义

### Stage 1: 构思 (Ideation)

```
阶段目标：
• 明确要解决的问题
• 验证问题是否真实存在
• 构思初步解决方案
• 评估可行性和价值

阶段产出：
□ 问题陈述文档
□ 市场规模估算
□ 初步解决方案构想
□ 竞品分析
□ 团队组建或确认

阶段时长：1-4周
成功标准：问题真实 + 方案可行 + 团队就位
```

### Stage 2: 验证 (Validation)

```
阶段目标：
• 开发MVP
• 测试核心假设
• 收集用户反馈
• 验证商业模式

阶段产出：
□ MVP产品
□ 首批测试用户（10-50人）
□ 用户反馈报告
□ 假设验证结论
□ 迭代方向确认

阶段时长：2-8周
成功标准：用户愿意使用 + 核心假设验证通过
```

### Stage 3: 开发 (Development)

```
阶段目标：
• 完成产品开发
• 迭代优化功能
• 建立技术壁垒
• 准备规模化

阶段产出：
□ 产品V1.0
□ 技术文档
□ 运营SOP
□ 首批付费用户
□ 规模化方案

阶段时长：2-6个月
成功标准：产品稳定 + 用户增长 + 初步盈利
```

### Stage 4: 测试 (Testing)

```
阶段目标：
• 规模化推广测试
• 优化获客成本
• 验证单位经济模型
• 打磨运营流程

阶段产出：
□ 增长数据报告
□ 获客成本分析
□ 单位经济模型
□ 运营SOP完善版
□ A轮材料准备

阶段时长：1-3个月
成功标准：LTV/CAC > 3 + 月增长 > 20%
```

### Stage 5: 发布 (Launch)

```
阶段目标：
• 正式发布产品
• 大规模市场推广
• 完成融资
• 建立品牌

阶段产出：
□ 正式发布
□ PR传播
□ A轮融资完成
□ 团队规模化
□ 行业影响力

阶段时长：持续
成功标准：行业认可 + 稳定增长 + 品牌建立
```

### Stage 6: 完成 (Completion)

```
阶段目标：
• 项目成功退出或转化
• 沉淀经验教训
• 资产变现
• 团队安置

阶段产出：
□ 退出方案（IPO/并购/转型）
□ 财务回报
□ 案例复盘文档
□ 团队安置方案
□ 联盟贡献

阶段时长：按需
成功标准：达到预期收益目标
```

## 五、项目运作指南

### 项目启动流程

```
1️⃣ 创建项目
   ↓
2️⃣ 填写基础信息（Name, Goal, Stage, Owner）
   ↓
3️⃣ 关联 Agents（至少1个）
   ↓
4️⃣ 设定里程碑（3-5个关键节点）
   ↓
5️⃣ 创建首批任务
   ↓
6️⃣ 启动项目
```

### 每日运作

```
🌅 Morning (5分钟)
1. 查看项目 Dashboard
2. 确认今日任务
3. 更新任务进度

🕐 Working (实时)
1. 执行任务
2. 记录发现到 Memory
3. 更新项目进度

🌙 Evening (5分钟)
1. 更新里程碑状态
2. 记录阻塞问题
3. 规划明日工作
```

### 阶段转换

```
阶段完成条件：
□ 阶段产出全部完成
□ 阶段评审通过
□ 下一个阶段准备就绪

阶段转换步骤：
1. 填写阶段总结
2. 更新 Stage
3. 创建下一阶段里程碑
4. 分配资源
5. 通知相关人
```
